# 更新日志

## 日志书写规范

1. **版本号记录规则**：
   - 只在**更新版本号的那次更新**后记录新版本号
   - 之后的**日常更新无需重复记录版本号**
   - 直至**下一次版本号更新时**再进行记录

2. **日期格式**：使用 `YYYY-MM-DD` 格式，如 `2025-12-27`

3. **内容格式**：
   - 每条更新记录以 `-` 开头
   - 简短描述更新内容，使用中文
   - 对于重大更新，可添加 `[重磅更新]` 标记

4. **排序规则**：更新记录按**日期从新到旧**排列

5. **提交记录**：
   - 包含 Git 提交哈希、作者、日期和提交信息
   - 格式：`- <哈希> <作者> <日期> <提交信息>`
